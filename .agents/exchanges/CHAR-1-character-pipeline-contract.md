## Request

Accept the shared integration boundary for the approved generated-character pipeline
before isolated character results move into shared runtime implementation.

Requested contract:

- Character Generation & Animation owns isolated humanoid/quadruped body, rig, skinning,
  physical-profile, motion, reaction, recovery, terrain-contact, evaluation, and
  Blender-to-Unreal adapter experiments.
- Blender owns authoritative offline source generation and tests; Unreal owns native live
  floor, IK, physics, reaction, runtime authority, and performance behavior.
- One downstream contract serves humanoid and quadruped family adapters.
- Future runtime variation borrows UE 5.8 Mutable inside prepared body-plan families;
  new anatomy remains an offline generated/cooked-content operation.
- Verified isolated intake evidence is offered through
  [`CHAR-2`](CHAR-2-unreal-humanoid-intake.md); shared adoption remains owned by Primary
  Builder.
- Primary Builder retains shared Unreal integration and `research/machines.json` edits.
  Verified Character output will return through a later `INTEGRATE` packet.

## Response

Accepted with the editor/runtime boundary recorded in `CHAR-2`.

- Character Generation & Animation retains ownership of isolated bodies, rigs,
  skinning, physical profiles, motion, reactions, recovery, terrain contact, evaluation,
  and Blender-to-Unreal adapter experiments.
- Blender remains the offline source-generation authority; Unreal owns live floor, IK,
  physics, reactions, runtime authority, and performance behavior.
- Prepared body-plan families may later use Mutable for bounded runtime variation; new
  anatomy remains an offline generated/cooked-content operation.
- Primary Builder retains shared Unreal runtime integration and registry authority.
- Verified specialist output returns through an `INTEGRATE` packet; `CHAR-2` is accepted
  as topology evidence without prematurely integrating or promoting it.

## Resolution

Accepted. Character Generation & Animation will continue isolated editor/cook-time
experiments and return verified outputs through integration packets. Shared runtime code
will consume cooked assets plus versions/provenance; Primary Builder retains runtime
bridge and machine-registry authority.
